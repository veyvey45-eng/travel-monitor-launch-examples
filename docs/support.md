# Support

This document describes the public support path for **Hotel Rate Monitoring API** and its related examples repository.

## Primary support channel

The primary support channel for the public repository assets is the GitHub issue tracker:

> https://github.com/veyvey45-eng/travel-monitor-launch-examples/issues

Use that channel when you need help with:

| Topic | Best use of the support channel |
|---|---|
| **Repository issues** | Broken links, documentation gaps, incorrect examples, missing assets |
| **Quickstart problems** | Problems reproducing the documented first-success flow |
| **MCP manifest questions** | Clarifications about `server.json`, registry visibility, or client setup |
| **Sample payload questions** | Questions about the published examples and expected field meanings |

## What to include in a support request

To make triage faster, support requests should include a short problem summary, the exact step that failed, the URL or file being used, the MCP client or HTTP tool involved, and any non-sensitive error message.

## Sensitive information

Do **not** post secrets in public support requests. This includes API keys, Apify tokens, hotel partner credentials, or customer data. Redact secrets before sharing logs or screenshots.

## Scope of support

This repository is intended to help users reach a first successful evaluation quickly. Support for the repository focuses on public documentation, onboarding clarity, example payloads, and basic MCP/API usage guidance.

Platform-level account, billing, or infrastructure issues may need to be handled with the corresponding platform provider.
